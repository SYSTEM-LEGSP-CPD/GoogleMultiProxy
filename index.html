Google全站反向代理多平台部署指南

支持Cloudflare Workers、Vercel、Netlify和Vite部署，实现Google全站代理及登录页面跳转到gov.cn功能

注意：本页面提供完整部署指南，包含所有平台的详细步骤和一键部署按钮

Cloudflare Workers 部署 推荐

环境变量配置

# 必填项
MAIN_DOMAIN=yourdomain.com
TARGET_ROOT=google.com
REDIRECT_URL=https://gov.cn

详细部署步骤

  1. 安装Wrangler CLI 
    npm install -g @cloudflare/wrangler
  2. 登录Cloudflare 
    wrangler login

    这将打开浏览器并引导您完成登录过程

  3. 配置wrangler.toml 
    name = "google-proxy"
    main = "src/worker.js"
    compatibility_date = "2025-10-29"

    [env.production]
    account_id = "your_account_id"
    api_token = "your_api_token"
    workers_dev = false
    route = "*.yourdomain.com/*"
    zone_id = "your_zone_id"

    将your_account_id和your_api_token替换为您的实际值

  4. 部署Worker 
    wrangler deploy src/worker.js \
      --env production \
      --vars "{\"MAIN_DOMAIN\":\"yourdomain.com\"}"
  5. 配置DNS记录 
    *.yourdomain.com   CNAME   yourdomain.com   代理状态：橙色云

    在Cloudflare DNS设置中添加上述CNAME记录

提示：Cloudflare Workers是本方案的推荐部署方式，因为它提供最快的响应速度和最完整的功能支持。

Vercel 部署

环境变量配置

# 必填项
MAIN_DOMAIN=yourdomain.com
TARGET_ROOT=google.com
REDIRECT_URL=https://gov.cn

详细部署步骤

  1. 安装Vercel CLI 
    npm install -g vercel
  2. 创建环境变量文件 
    echo "MAIN_DOMAIN=yourdomain.com" > .env
    echo "TARGET_ROOT=google.com" >> .env
    echo "REDIRECT_URL=https://gov.cn" >> .env
  3. 部署到Vercel 
    vercel --prod

    按照提示完成部署过程

  4. 配置自定义域名 
    vercel domains add yourdomain.com

    按照提示完成域名配置

  5. 配置DNS记录 
    *.yourdomain.com   CNAME   cname.vercel-dns.com

    在您的DNS提供商处添加上述CNAME记录

提示：Vercel提供免费的Edge Functions服务，适合需要全球CDN加速的场景。

环境变量配置

# 必填项

TARGET_ROOT=google.com
REDIRECT_URL=https://gov.cn

详细部署步骤

  1. 登录Netlify控制台

    访问 https://app.netlify.com/ 并使用GitHub账户登录

  2. 创建新站点

    点击"New site from Git"按钮

  3. 选择GitHub仓库

    选择包含本项目的GitHub仓库

  4. 配置构建设置 
    Build command: npm run build
    Publish directory: dist
  5. 设置环境变量

    在"Environment"部分添加以下环境变量：

    • MAIN_DOMAIN=yourdomain.com
    • TARGET_ROOT=google.com
    • REDIRECT_URL=https://gov.cn
  6. 完成部署

    点击"Deploy site"按钮完成部署

  7. 配置自定义域名

    在"Domain management"中添加您的自定义域名

提示：Netlify提供28秒的函数执行时间，比Cloudflare和Vercel的10秒更长，适合需要更长时间处理的场景。

Vite 静态部署 文档/简单UI

环境变量配置

# 必填项（以VITE_开头）
VITE_MAIN_DOMAIN=yourdomain.com
VITE_TARGET_ROOT=google.com
VITE_REDIRECT_URL=https://gov.cn

详细部署步骤

  1. 安装Vite 
    npm install -g create-vite
  2. 创建新项目 
    create-vite my-proxy --template vanilla
  3. 替换项目文件

    将本仓库中的以下文件复制到新项目中：

    • src/main.js
    • src/style.css
    • index.html
    • vite.config.js
    • docs/ (整个目录)
  4. 安装依赖 
    cd my-proxy
    npm install
  5. 开发模式 
    npm run dev

    在浏览器中访问 http://localhost:5173 查看效果

  6. 构建生产版本 
    npm run build

    生成的文件将位于dist/目录

  7. 部署到静态托管

    使用一键部署按钮部署到Netlify/Vercel，或手动将dist/目录上传到您的静态托管服务

提示：Vite主要用于文档网站和简单UI，不适用于完整的代理功能。它适合展示本项目的文档和部署指南。

❓ 常见问题

Q: 为什么我的代理无法访问Google服务？

A: 可能是Google检测到代理请求并触发了CAPTCHA。尝试以下解决方案：

  • 添加User-Agent过滤，只允许常见浏览器
  • 配置合理的请求频率限制
  • 使用Cloudflare的IP池，避免单一IP被封禁

Q: 如何自定义CSP（内容安全策略）？

A: 在worker.js中修改CSP_POLICY配置：

CSP_POLICY: {
'default-src': "'self'",
'script-src': "'self' 'unsafe-inline' 'unsafe-eval'",
'style-src': "'self' 'unsafe-inline'",
'img-src': "'self' data: https://*.google.com",
'connect-src': "'self' https://*.google.com",
'font-src': "'self'",
'object-src': "'none'",
'media-src': "'self'"
}

Q: 如何添加更多需要跳转的路径？

A: 在worker.js中修改LOGIN_PATHS配置：

LOGIN_PATHS: [
'/login',
'/signin',
'/signup',
'/register',
'/account',
'/auth',
'/user/login'
]

Q: 部署后出现403错误怎么办？

A: 403错误通常由以下原因引起：

  • Cloudflare的防火墙规则阻止了请求 - 检查防火墙设置
  • 目标网站（Google）阻止了代理请求 - 尝试添加User-Agent头
  • DNS记录未正确配置 - 确保CNAME记录已设置且为代理状态（橙色云）
